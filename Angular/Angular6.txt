----- TERMINAL -----
Usando o Angular - > Necessário ter instalado o node e npm
Gerando arquivos a partir do Angular CLI - > Terminal

    INSTALANDO TODAS AS DEPENDENCIAS DE UM PROJETO
$npm install

    PLUGINS VISUAL STUDIO CODE
Auto-Import
Bracket Pair Colorized

    O NPM INSTALA O ANGULAR.
$sudo npm install -g @angular/cli

    O NPM INSTALA O TYPESCRIPT
$sudo npm install -g typescript
$tsc -v

    CONVERTEO TYPESCRIPT PARA JAVASCRIPT
$tsc main.ts
Caso haja campos get e set
$tsc *.ts --target ES6 && node main.js 

    NOVO PROJETOS
$ng new Nome_da_Applicação

    CRIAMOS UM SERVER DA APLICAÇÃO
$ng serve -o

  CRIA UM NOVO COMPONENTE NA PASTA ATUAL
$ng g c Nome_do_Componente 
$ng generate component Nome_do_Componente 

    CRIA UM NOVO SERVIÇO NA PASTA ATUAL  
$ng g s Nome_do_Serviço

   ADICIONANDO O BOOTSTRAP AO ANGULAR
$npm install bootstrap --save DENTRO DO PROJETO

----- PASTAS -----
e2e (end 2 end)
Aqui é uma pasta que ao escrever código simula o nosso cliente/usuário .Esta pasta é especifica para criar e 
fazer testes.

node_modules
Pasta que estoca/registra/armazena pacotes de terceitos para o desenvolvimento da aplicação.
Ou seja bibliotecas de terceiros que enriquecem o desenvolvimento.

src/app
Cada aplicação tem no minimo um módulo e um componente.

src/assets
Patrimônio histórico da sua aplicação.

src/environments
Configurações de ambiente do projeto,para diferentes ambientes , um para produção e outro para local

/main.ts
Ponto inicial da aplicação do projeto,

/polyfills.ts
Aquivo que carrega scripst para o funcionamento do angular no navegador.

/.editorconfig
Caso seu projeto seja em equipe aqui você determina regras globais para seus editores,quando eles estiverem
a usar a aplicação.

/.gitignore
Arquivo de regras que ao push e pull do git você ignora alguns arquivos.
